# Media Cleaner для macOS

`media_cleaner.py` — локальный Python-скрипт для безопасного сжатия фото и видео, поиска точных дубликатов и переноса лишних копий в отдельную папку.

Скрипт работает на вашем Mac. Он не загружает файлы в интернет и не удаляет оригиналы автоматически.

## Возможности

- Рекурсивно обрабатывает папки вместе со всеми вложенными папками.
- Сжимает видео в MP4 с кодеком HEVC (`libx265`) и совместимым с QuickTime тегом `hvc1`.
- Сжимает JPEG, стараясь сохранить EXIF и цветовые данные.
- Копирует PNG, HEIC, RAW, документы и неизвестные файлы без изменений.
- Находит точные дубликаты по размеру и SHA256.
- Может безопасно переместить лишние дубликаты в `Duplicates_To_Delete`.
- Поддерживает dry-run без создания, копирования или сжатия файлов.
- Не перезаписывает существующие результаты и отчёты.

## Требования

- macOS
- Python 3
- ffmpeg
- Pillow для обработки JPEG

Проверить установку:

```bash
python3 --version
ffmpeg -version
python3 -c "import PIL; print(PIL.__version__)"
```

Если ffmpeg не установлен:

```bash
brew install ffmpeg
```

Если Pillow не установлен:

```bash
python3 -m pip install Pillow
```

## Рекомендуемый запуск

Откройте Terminal, перейдите в папку проекта и запустите мастер:

```bash
cd "/Users/floraison/MediaCleanerProject"
python3 media_cleaner.py --wizard
```

Мастер предложит:

1. Только найти дубликаты.
2. Сделать dry-run сжатия папки.
3. Реально сжать папку.
4. Быстро сжать один видеофайл.
5. Безопасно убрать лишние дубликаты.
6. Выйти.

Путь к файлу или папке можно вставить вручную либо перетащить объект из Finder в окно Terminal.

## Сжатие папки

Сначала рекомендуется выполнить dry-run:

```bash
python3 media_cleaner.py "/путь/input" "/путь/output" --dry-run
```

Реальная обработка:

```bash
python3 media_cleaner.py "/путь/input" "/путь/output"
```

Настройка качества:

```bash
python3 media_cleaner.py "/путь/input" "/путь/output" \
  --video-crf 31 \
  --video-preset fast \
  --image-quality 85
```

Показать обычный прогресс ffmpeg с `frame`, `fps`, `time`, `bitrate` и `speed`:

```bash
python3 media_cleaner.py "/путь/input" "/путь/output" --show-ffmpeg-progress
```

## Быстрое сжатие одного видео

```bash
python3 media_cleaner.py \
  --quick-file "/путь/video.mov" \
  --video-crf 31 \
  --video-preset fast
```

Результат сохраняется рядом с оригиналом. Например:

```text
IMG_1234.MOV
IMG_1234_compressed_crf31_fast.mp4
```

Если такое имя уже занято, скрипт добавит `_2`, `_3` и далее. Оригинальный файл не изменяется.

## Поиск дубликатов

Только найти точные дубликаты и создать отчёты:

```bash
python3 media_cleaner.py "/путь/input" "/путь/output" --duplicates-only
```

Дубликаты определяются безопасно:

1. Файлы группируются по размеру.
2. SHA256 вычисляется только для файлов одинакового размера.
3. Совпадение SHA256 означает точный дубликат.

Этот режим ничего не перемещает и не удаляет.

## Безопасный перенос дубликатов

Сначала выполните dry-run:

```bash
python3 media_cleaner.py "/путь/input" "/путь/output" \
  --move-duplicates --dry-run
```

Реальный перенос:

```bash
python3 media_cleaner.py "/путь/input" "/путь/output" --move-duplicates
```

Первая копия каждого файла остаётся на месте. Лишние копии перемещаются с сохранением структуры папок в:

```text
output/Duplicates_To_Delete/
```

Ничего не удаляется навсегда. Папку `Duplicates_To_Delete` пользователь проверяет и удаляет вручную только при полной уверенности.

## Качество видео

`--video-crf` принимает значения от `18` до `35`:

- меньше число — выше качество и больше файл;
- больше число — сильнее сжатие и меньше файл;
- значение по умолчанию — `31`.

`--video-preset` управляет скоростью кодирования:

```text
ultrafast, superfast, veryfast, faster, fast, medium, slow
```

По умолчанию используется `fast`. Более медленный preset обычно сжимает эффективнее, но обработка занимает больше времени.

## Качество JPEG

Параметр `--image-quality` принимает значения от `1` до `100`. По умолчанию используется `85`.

Пример:

```bash
python3 media_cleaner.py input output --image-quality 90
```

## Отчёты

Отчёты создаются в папке:

```text
output/reports/
```

В зависимости от режима там могут находиться:

- `summary.csv` — общая сводка;
- `duplicates_report.csv` — группы точных дубликатов;
- `moved_duplicates.csv` — список перемещённых или запланированных к перемещению копий;
- `errors.log` — ошибки обработки.

Существующие отчёты не перезаписываются. Для нового запуска лучше использовать новую output-папку.

## Правила безопасности

- Input и output должны быть разными папками.
- Output нельзя размещать внутри input.
- Оригинальная input-папка не меняется при обычном сжатии.
- Существующие файлы не перезаписываются.
- При переносе дублей минимум одна копия остаётся на месте.
- Постоянного автоматического удаления файлов в программе нет.
- Перед большой обработкой рекомендуется использовать `--dry-run` и тестовую папку.
