# Media Cleaner

Локальный Python-скрипт для наведения порядка в медиаархиве: сжатие фото и видео, поиск точных и визуально похожих дубликатов, а также пересборка фильмов и сериалов в MKV с выбором аудиодорожек и субтитров — в том числе **без перекодирования видео**.

Скрипт работает полностью локально. Ничего не загружается в интернет, оригиналы никогда не изменяются и не удаляются автоматически.

## Возможности

**Дубликаты**
- Поиск точных дубликатов по размеру и SHA256 с отчётами в CSV.
- Безопасный перенос лишних копий в папку `Duplicates_To_Delete` (минимум одна копия всегда остаётся на месте).
- HTML-обзор дубликатов с миниатюрами для ручной проверки.

**Похожие файлы**
- Поиск визуально похожих фото по перцептивному хешу (pHash) с настраиваемой строгостью.
- Поиск визуально похожих видео по хешам выборочных кадров.
- Перемещение отобранных кандидатов в корзину (только macOS).

**Сжатие**
- Пакетное сжатие архива: видео в HEVC с тегом `hvc1`, JPEG с сохранением EXIF.
- PNG, HEIC, RAW, документы и неизвестные файлы копируются без изменений.
- Быстрое сжатие одного видеофайла.
- Режим `--dry-run`: показывает план, ничего не создавая.

Про выбор кодека стоит знать одну вещь. Значение по умолчанию `--video-codec x265` — это **автовыбор**, а не гарантированный `libx265`: на macOS будет использован аппаратный `hevc_videotoolbox`, если ffmpeg его поддерживает, и только иначе `libx265`. Аппаратный кодек в разы быстрее, но управляется параметром `q:v`, в который пересчитывается CRF, поэтому точность CRF теряется. Если нужен именно `libx265` с точным CRF, добавьте `--disable-hw-video` (в мастере — отдельный пункт «libx265 (только процессор)»). Мастер показывает реальный кодек в блоке «Итоговые настройки» ещё до подтверждения.

**Фильмы и сериалы (MKV)**
- Выбор конкретных видео-, аудиодорожек и субтитров с понятным описанием каждой (язык, кодек, число каналов, название).
- Режим **без сжатия** — видео копируется байт в байт (`-c:v copy`), меняется только набор дорожек. Быстро и без потери качества.
- Либо перекодирование в `libx265` с выбором CRF и preset.
- Либо **быстрое сжатие аппаратным кодеком** — примерно втрое быстрее. Пункт появляется в меню, только если такой кодек есть на этой машине и прошёл реальную проверку. Качество задаётся целевым битрейтом, а мастер сразу показывает ожидаемый размер файла.

Про аппаратное сжатие честно: оно кратно быстрее, но при одинаковом размере файла качество заметно ниже, чем у `libx265`. Поэтому по умолчанию предлагается не оно. Посмотреть, что доступно на вашей машине: `mc --list-encoders`. Аппаратное ускорение проверено автором только на Apple Silicon; ветки для NVIDIA, Intel и AMD написаны вслепую, выбираются только после реальной тестовой попытки кодирования и при любой ошибке безопасно откатываются на `libx265`.
- Можно добавить **несколько файлов сразу**. Если раскладка дорожек совпадает (обычные серии одного сезона), номера спрашиваются **один раз** и применяются ко всем файлам.

## Требования

- Python 3.9 или новее (разрабатывалось и проверялось на 3.14).
- [ffmpeg](https://ffmpeg.org/) вместе с `ffprobe` — для всего, что касается видео.
- `pillow` и `imagehash` — ставятся автоматически вместе с программой; используются только для поиска похожих файлов.
- `pillow-heif` — опционально, для чтения HEIC. Ставится отдельно, командой с extra `heic` (см. установку).

## Установка на macOS

**1. Python.** Проверьте, установлен ли он:

```bash
python3 --version
```

Если команда не найдена, скачайте Python с [python.org](https://www.python.org/downloads/macos/) и установите. Затем закройте и снова откройте Terminal.

**2. ffmpeg.** Проще всего через [Homebrew](https://brew.sh/):

```bash
brew install ffmpeg
```

**3. Сам Media Cleaner.** Рекомендуется через [pipx](https://pipx.pypa.io/) — он ставит программу в изолированное окружение, но команда работает из любой папки:

```bash
brew install pipx
pipx ensurepath
pipx install git+https://github.com/ilya-konoplev/media-cleaner.git
```

Если нужна поддержка HEIC, ставьте вариант с extra `heic`:

```bash
pipx install "media-cleaner[heic] @ git+https://github.com/ilya-konoplev/media-cleaner.git"
```

Закройте и снова откройте Terminal.

**4. Готово.** Теперь в любой папке достаточно набрать:

```bash
mc
```

Откроется пошаговый мастер. Больше ничего запоминать не нужно.

## Установка на Windows

**1. Python.** Скачайте с [python.org](https://www.python.org/downloads/windows/). При установке обязательно отметьте галочку **«Add python.exe to PATH»**. Проверьте в PowerShell:

```powershell
python --version
```

**2. ffmpeg.** Через встроенный менеджер пакетов:

```powershell
winget install Gyan.FFmpeg
```

Затем закройте и снова откройте PowerShell и проверьте:

```powershell
ffmpeg -version
```

Если `winget` недоступен, скачайте сборку с [ffmpeg.org](https://ffmpeg.org/download.html), распакуйте и добавьте папку `bin` в переменную среды `PATH`.

**3. Сам Media Cleaner:**

```powershell
python -m pip install --user pipx
python -m pipx ensurepath
pipx install git+https://github.com/ilya-konoplev/media-cleaner.git
```

Если нужна поддержка HEIC, ставьте вариант с extra `heic`:

```powershell
pipx install "media-cleaner[heic] @ git+https://github.com/ilya-konoplev/media-cleaner.git"
```

Закройте и снова откройте PowerShell.

**4. Готово.** Теперь в любой папке достаточно набрать:

```powershell
mc
```

Откроется пошаговый мастер.

> Для установки нужен ещё [git](https://git-scm.com/download/win). Если ставить его не хочется, скачайте репозиторий кнопкой **Code → Download ZIP**, распакуйте и выполните `pipx install C:\путь\к\папке`.

### Что на Windows не работает

Основное — дубликаты, похожие файлы, сжатие и режим MKV — работает одинаково на обеих системах. Но часть возможностей завязана на macOS:

| Функция | Windows |
| --- | --- |
| Перемещение в корзину (пункты 8 и 10) | Недоступно, скрипт сообщит об ошибке. Отчёты создаются, файлы можно удалить вручную |
| Аппаратный кодек `hevc_videotoolbox` | Недоступно, используйте `x265` |
| Автооткрытие папки с результатом | Недоступно, откройте папку вручную |

Ещё одно отличие — ввод путей. Перетаскивание файлов в окно терминала это возможность macOS Terminal; в Windows пути вставляют вручную. Обратные слэши при этом вставляются как есть, ничего экранировать не нужно: `C:\Users\Ilya\Video.mkv` вводится ровно так. Несколько путей разделяются пробелом, а путь с пробелами заключается в двойные кавычки: `"C:\Users\Ilya\My Video.mkv"`. Один путь с пробелами можно вставить и без кавычек — скрипт распознает его целиком, если такая папка или файл существует.

## Использование

Просто наберите в терминале:

```bash
mc
```

Откроется пошаговый мастер с 12 пунктами: поиск дубликатов, сжатие, поиск похожих файлов, работа с фильмами и сериалами. Флаги знать не нужно — мастер обо всём спросит сам и в конце покажет итоговые настройки перед запуском.

### Примеры для командной строки

Если хочется без мастера, почти все режимы доступны флагами. Исключение — пересборка в MKV (пункт 11): она работает только через мастер, потому что требует интерактивного выбора дорожек.

```bash
# Посмотреть план, ничего не меняя
mc ~/Photos ~/Photos_compressed --dry-run

# Реальное сжатие
mc ~/Photos ~/Photos_compressed

# Только найти точные дубликаты
mc ~/Photos ~/Report --duplicates-only

# Перенести лишние копии (сначала с dry-run)
mc ~/Photos ~/Cleanup --move-duplicates --dry-run

# Найти похожие фото построже
mc ~/Photos ~/Review --find-similar-photos --similar-threshold 3

# Быстро сжать один файл
mc --quick-file ~/Movies/clip.mov --video-crf 31 --video-preset fast

# Показать, какие кодеки доступны на этой машине
mc --list-encoders

# Перепроверить кодеки заново, игнорируя кэш
mc --list-encoders --refresh-hw-cache
```

Результат проверки кодеков кэшируется в `~/.cache/media_cleaner/`, чтобы не гонять её при каждом запуске. Кэш привязан к версии ffmpeg и сам сбрасывается после его обновления, так что `--refresh-hw-cache` нужен редко.

Полный список параметров:

```bash
mc --help
```

Если имя `mc` уже занято (например, файловым менеджером Midnight Commander), у программы есть второе имя — `media-cleaner`. Работает точно так же.

### Запуск без установки

Скрипт можно запускать и просто как файл, без пакета:

```bash
python3 media_cleaner.py --wizard
```

## Работа с сериалами

Пункт 11 мастера рассчитан на сезон целиком. Проще всего перетащить **папку сезона** — скрипт возьмёт видеофайлы, лежащие непосредственно в ней.

> Вложенные папки по умолчанию не сканируются. Но если на верхнем уровне видео не нашлось, а во вложенных папках они есть, скрипт покажет список таких подпапок и спросит, искать ли в них. При согласии структура подпапок сохраняется в выходной папке, поэтому `Season 1/Ep01.mkv` и `Season 2/Ep01.mkv` не конфликтуют.

Результаты прошлых запусков скрипт узнаёт и не обрабатывает повторно — и по имени файла, и по имени папки (`S01 remux`, `S01 x265 crf22`). Совпадение суффикса проверяется точно, поэтому ваша собственная папка вроде `remux backup` останется видимой. Пропущенные файлы всегда перечисляются по именам: если это всё-таки ваши файлы, перетащите их по отдельности — поштучное перетаскивание фильтр не применяет.

> Перетаскивать сами файлы пачкой тоже можно, но терминал принимает не больше 1024 символов в одной строке. При длинных путях это примерно девять файлов, дальше символы просто перестают вводиться. Папка обходит это ограничение: её путь короткий независимо от числа серий.

Добавив файлы, вы увидите, что скрипт:

1. Проанализирует потоки каждого файла через `ffprobe`.
2. Сравнит раскладку дорожек. Сравнение идёт по индексу, типу, кодеку, языку и числу каналов; название дорожки игнорируется, поэтому серии с разными заголовками считаются одинаковыми.
3. Видео с аудио и субтитры сравнит **раздельно**. Если видео и аудио совпадают во всех файлах, их номера спрашиваются один раз. Субтитры тоже спрашиваются один раз, если совпадают, и по каждому файлу, если различаются — так бывает, когда в одной серии на дорожку субтитров меньше.
4. Настройки сжатия спросит один раз для всех файлов.
5. Обработает файлы по очереди. Ошибка на одном файле не прерывает остальные.

На шаге сжатия есть три принципиально разных варианта:

| Выбор | Скорость | Когда брать |
| --- | --- | --- |
| **БЕЗ СЖАТИЯ — copy** (по умолчанию) | меньше минуты | Нужно только убрать лишние дорожки. Размер не изменится, качество не пострадает |
| **Аппаратный кодек** | примерно втрое быстрее libx265 | Нужно уменьшить сезон и важно время. Качество при равном размере ниже, чем у libx265 |
| **libx265** | несколько часов на фильм | Нужно максимальное качество на единицу размера, время не критично |

Пункт с аппаратным кодеком появляется, только если такой кодек есть на этой машине и прошёл реальную проверку. Качество там задаётся целевым битрейтом, и мастер сразу показывает ожидаемый размер файла — выбирать приходится не вслепую. Мастер также напоминает о стоимости каждого варианта в блоке «Итоговые настройки».

Результаты разных режимов складываются в разные папки (`… remux`, `… hw hevc 1480k`, `… x265 crf22`), так что их можно сравнить между собой и не перепутать.

## Дополнительные возможности

Все они **выключены по умолчанию** и включаются своим флагом. Мастер и привычные режимы работают ровно как раньше, если флаги не указывать.

### Выбирать фото прямо в браузере

```bash
mc ~/Photos ~/Review --find-similar-photos      # сначала найти похожие
mc ~/Photos ~/Review --review-similar-photos --pick
```

Второй командой открывается страница со всеми группами: снимки лежат рядом крупно, у каждого радиокнопка «оставить этот». Отмечаете нужные, жмёте «Сохранить выбор» — и решение записывается в отчёт. Дальше `--trash-similar-from-report` отправит в корзину всё, кроме выбранного вами.

Страница обслуживается локальным помощником на `127.0.0.1` (браузер не может писать на диск из обычного файла). Наружу ничего не отправляется, помощник останавливается по Ctrl-C. Без `--pick` страница остаётся просто обзором.

### Подсказать самый резкий кадр серии

```bash
mc ~/Photos ~/Review --find-similar-photos --best-shot
```

Из серии почти одинаковых снимков «оставить» предлагается не первый по алфавиту, а самый резкий: считается детализация, учитываются разрешение и вес. В отчёт добавляются колонки `sharpness` и `best_shot_score`.

Ограничение, о котором стоит знать: сильно пережатый JPEG может получить завышенную оценку — артефакты сжатия похожи на мелкие детали. Поэтому воспринимайте это как подсказку и проверяйте глазами через `--pick`, а не как окончательный вердикт.

### Не пережимать то, что уже сжато

```bash
mc ~/Photos ~/Compressed --skip-already-compressed
```

Видео, уже лежащее в HEVC с разумным битрейтом, и фотографии, уже сохранённые с невысоким качеством, пропускаются и копируются как есть. Без этого флага такие файлы прогоняются через кодировщик впустую: время тратится, а результат бывает даже больше оригинала и хуже по качеству. Причина пропуска пишется в `summary.csv`.

### Ускорить поиск точных дубликатов

```bash
mc ~/Photos ~/Report --duplicates-only --fast-duplicates
```

Файлы одинакового размера сначала сравниваются по краям и только при совпадении дохешиваются целиком. На больших видео это заметно быстрее (на тестовых 14 ГБ — в 3.5 раза). Решение «дубликат» по-прежнему принимается только по полному SHA256, так что точность не страдает.

### Вернуть перемещённые дубликаты

```bash
mc ~/Photos ~/Cleanup --undo-move-duplicates --dry-run
mc ~/Photos ~/Cleanup --undo-move-duplicates
```

Отменяет `--move-duplicates`, возвращая файлы туда, откуда они уехали. Перед возвратом проверяется, что файл на месте, размер совпадает с записанным и исходное место свободно — ничего не перезаписывается.

## Безопасность

- Оригиналы никогда не изменяются и не удаляются.
- Результат всегда пишется в отдельную папку; input и output не могут находиться друг внутри друга.
- Запись идёт во временный файл и переставляется на место только после успешного завершения.
- Существующие файлы и отчёты не перезаписываются.
- Символические ссылки пропускаются.
- Единственное удаление — перемещение в корзину macOS, и только по явной команде из отчёта. Безвозвратного удаления в скрипте нет.

## Отчёты

Создаются в подпапке `reports` внутри output-папки:

- `summary.csv` — что и как было обработано, размеры до и после.
- `duplicates_report.csv` — найденные группы точных дубликатов.
- `similar_photos_report.csv`, `similar_videos_report.csv` — группы похожих файлов с пометкой `keep_candidate` или `review_duplicate`.
- `errors.log` — ошибки обработки.

Отчёты о похожих файлах предназначены для ручной проверки: сначала просмотрите их и только потом запускайте удаление в корзину.
